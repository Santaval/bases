# Report 1 — User 360

## Objetivo

Consolidar múltiples fuentes de datos en una visión holística e integral de cada socio. El reporte permite a operadores y administradores consultar el perfil completo de un usuario: historial de actividad, estado de membresía activa, nivel de engagement (ICC), evolución física, badges obtenidos y soporte solicitado.

---

Usar **Lookup Activity** sobre cada tabla Gold para verificar que `MAX(etl_run_date) >= @addDays(utcNow(), -1)`. Si falla, usar **If Condition** para emitir un fallo controlado con mensaje de error descriptivo.

## Paso 1 — Ingesta a Bronze (tablas complementarias)

### Copy_transactions
```sql
SELECT id, userID, activityID, value, membershipID, createdAt
FROM transactions
WHERE createdAt >= '@{pipeline().parameters.p_watermark_date}'
```

### Copy_users_tickets + Copy_tickets
```sql
-- users_tickets
SELECT user, ticket FROM users_tickets;

-- tickets
SELECT id, title, color FROM tickets;
```

### Copy_supportTickets + Copy_supportTicketComments
```sql
-- support tickets
SELECT id, title, status, priority, createdAt, updatedAt, user
FROM supportTickets
WHERE user IS NOT NULL;

-- comments
SELECT id, supportTicket, comment, createdAt, user
FROM supportTicketComments;
```

## Paso 2 — Silver: Preparación de Suplementos (`DF_Silver_User360_Supplements`)

### Stream A: `stm_user_memberships`

**Source** → `Bronze_users_memberships` JOIN `Bronze_memberships`

1. **Join** on `users_memberships.membership = memberships.id`
2. **Aggregate** — por `user`
   ```
   active_memberships     = collect(name)   -- lista de nombres de membresía
   membership_count       = count(membership)
   latest_membership_name = last(name)      -- orden por memberships.createdAt
   ```
3. **Sink** → `Silver_user_memberships_summary`

### Stream B: `stm_transactions_summary`

**Source** → `Bronze_transactions`

1. **Aggregate** — por `userID`
   ```
   total_transactions    = count(id)
   total_spent           = sum(value)
   first_transaction     = min(createdAt)
   last_transaction      = max(createdAt)
   unique_memberships_bought = countDistinct(membershipID)
   ```
2. **Sink** → `Silver_transactions_summary`

### Stream C: `stm_badges`

**Source** → `Bronze_users_tickets` JOIN `Bronze_tickets`

1. **Join** on `users_tickets.ticket = tickets.id`
2. **Aggregate** — por `user`
   ```
   badge_count  = count(ticket)
   badge_names  = collect(title)
   ```
3. **Sink** → `Silver_badges_summary`

### Stream D: `stm_support`

**Source** → `Bronze_supportTickets`

1. **Aggregate** — por `user`
   ```
   total_tickets       = count(id)
   open_tickets        = countIf(status == 'open')
   closed_tickets      = countIf(status == 'closed')
   last_ticket_date    = max(createdAt)
   high_priority_count = countIf(priority == 'high')
   ```
2. **Sink** → `Silver_support_summary`

## Paso 3 — Gold: Consolidación User 360 (`DF_Gold_User360`)

**Source principal** → `Bronze_users` (todos los usuarios activos)

1. **Filter** — `banned = 0 AND verified = 1`

2. **Left Join** con `dim_user_status` (Gold de Dashboard 1) on `users.id = user_id`
   - Columnas: `user_status`, `days_since_last_activity`, `last_activity_date`

3. **Left Join** con `fact_engagement_monthly` (Gold de Dashboard 2) — última fila por usuario
   - Usar **Window** `rank() over (partitionBy: user_id, orderBy: period_ym DESC) = 1`
   - Columnas: `icc_score`, `engagement_segment`, `monthly_attendance`, `period_ym AS last_engagement_period`

4. **Left Join** con `fact_progress_summary` (Gold de Dashboard 3) on `users.id = user_id`
   - Columnas: `overall_trend`, `total_evals`, `last_eval_date`, `total_fat_change`, `total_muscle_change`

5. **Left Join** con `Silver_user_memberships_summary` on `users.id = user`
   - Columnas: `latest_membership_name`, `membership_count`

6. **Left Join** con `Silver_transactions_summary` on `users.id = userID`
   - Columnas: `total_transactions`, `total_spent`, `first_transaction`, `last_transaction`

7. **Left Join** con `Silver_badges_summary` on `users.id = user`
   - Columnas: `badge_count`

8. **Left Join** con `Silver_support_summary` on `users.id = user`
   - Columnas: `total_tickets`, `open_tickets`, `high_priority_count`

9. **Derived Column** — `days_as_member`
   ```
   dateDiff('day', toDate(createdAt), currentDate())
   ```

10. **Derived Column** — `risk_flag`
    ```
    iif(user_status == 'at_risk' || user_status == 'churned'
        || engagement_segment == 'Low' || engagement_segment == 'Inactive'
        || open_tickets > 0, true(), false())
    ```

11. **Select** — columnas finales del reporte (ver esquema abajo)

12. **Sink** → `DS_AzureSQL_report_user_360`
    - **Write method**: `UPSERT` usando `user_id` como clave

## Esquema Gold (Output Table)

### `report_user_360`
```sql
CREATE TABLE report_user_360 (
    -- Identificación
    user_id                  VARCHAR(20)   NOT NULL,
    email                    VARCHAR(100)  NOT NULL,
    name                     VARCHAR(100),
    surname1                 VARCHAR(100),
    cedula                   VARCHAR(20),
    gender_label             VARCHAR(10),
    age                      INT,
    registration_date        DATETIME      NOT NULL,
    days_as_member           INT           NOT NULL,

    -- Estado de retención
    user_status              VARCHAR(10),   -- active|at_risk|churned
    last_activity_date       DATETIME,
    days_since_last_activity INT,

    -- Membresía
    latest_membership_name   VARCHAR(100),
    membership_count         INT           NOT NULL DEFAULT 0,

    -- Engagement
    last_engagement_period   VARCHAR(7),
    icc_score                DECIMAL(5,2),
    engagement_segment       VARCHAR(10),
    monthly_attendance       INT,

    -- Progreso físico
    overall_trend            VARCHAR(10),   -- Mejora|Estable|Declive
    total_evals              INT           NOT NULL DEFAULT 0,
    last_eval_date           DATE,
    total_fat_change         DECIMAL(6,2),
    total_muscle_change      DECIMAL(6,2),

    -- Transacciones
    total_transactions       INT           NOT NULL DEFAULT 0,
    total_spent              INT           NOT NULL DEFAULT 0,
    first_transaction_date   DATETIME,
    last_transaction_date    DATETIME,

    -- Badges
    badge_count              INT           NOT NULL DEFAULT 0,

    -- Soporte
    total_support_tickets    INT           NOT NULL DEFAULT 0,
    open_support_tickets     INT           NOT NULL DEFAULT 0,
    high_priority_tickets    INT           NOT NULL DEFAULT 0,

    -- Flag de riesgo
    risk_flag                BIT           NOT NULL DEFAULT 0,

    etl_run_date             DATETIME      DEFAULT GETDATE(),
    PRIMARY KEY (user_id)
);
```
